# Topic Blueprint Review Checklist

Use this checklist before approving any topic blueprint for question generation or reseeding.

## Coverage Check

- topic scope is clearly defined
- overlap topics are clearly excluded
- learning objectives are visible and balanced
- all major sub-concepts are represented

## Variety Check

- at least 5 different archetypes are planned
- assertion-reason is not the only advanced style
- application and case-based items are present where suitable
- short-answer is included when the topic can support it
- stems are not all variations of one sentence pattern

## Anti-Repetition Check

- repeated number-swapping templates are explicitly banned
- repeated sentence skeletons are explicitly banned
- repeated distractor patterns are explicitly banned
- one sub-concept is not being reused excessively
- the blueprint explains how to detect near-duplicates

## Difficulty Check

- foundation is not too trivial
- intermediate is not just bigger numbers
- advanced requires reasoning, elimination, inference, or concept linking
- advanced questions are not all of one type

## Sellability Check

- questions can feel premium and exam-ready
- contexts are realistic and varied
- distractors represent believable student mistakes
- explanations are expected to be concise and useful

## Implementation Check

- blueprint can be mapped into seed logic
- blueprint can be used by a manual content writer
- file naming is predictable
- class and subject placement are correct

## Red Flags

Reject or rewrite the blueprint if:

- one template can generate most of the pack
- “variety” is claimed but not operationalized
- the topic file has no banned patterns section
- the topic file does not separate foundation, intermediate, and advanced behavior
- the file could still allow 20 near-identical questions
